`create rune-app` コマンドを実行していくつかの質問に回答することにより、新しい Rune プロジェクトを素早く作成できます。

## 前提条件

- [Node.js](https://nodejs.org/) - `v22.12.0` 以上
- ターミナル - Rune プロジェクトの初期化にはコマンドライン操作が必要です

## インストール

<Steps>

1. **新しいプロジェクトの作成**

   ターミナルで以下のコマンドを実行し、新しい Rune プロジェクトを作成します。

   <Tabs syncKey="pkg">
     <TabItem label="pnpm">
     ```sh
     pnpm create rune-app
     ```
     </TabItem>
     <TabItem label="npm">
     ```sh
     npm create rune-app@latest
     ```
     </TabItem>
     <TabItem label="yarn">
     ```sh
     yarn create rune-app
     ```
     </TabItem>
     <TabItem label="bun">
     ```sh
     bun create rune-app
     ```
     </TabItem>
   </Tabs>

   プロジェクト名の入力、依存関係のインストール、git リポジトリの初期化について対話形式で確認されます。

   プロジェクト名は引数として直接渡すこともできます。

   <Tabs syncKey="pkg">
     <TabItem label="pnpm">
     ```sh
     pnpm create rune-app my-cli
     ```
     </TabItem>
     <TabItem label="npm">
     ```sh
     npm create rune-app@latest my-cli
     ```
     </TabItem>
     <TabItem label="yarn">
     ```sh
     yarn create rune-app my-cli
     ```
     </TabItem>
     <TabItem label="bun">
     ```sh
     bun create rune-app my-cli
     ```
     </TabItem>
   </Tabs>

   プロジェクト名の代わりに `.` を渡すと、現在のディレクトリにプロジェクトを作成できます。テンプレートと衝突するファイル（`package.json`、`src/` など）が含まれているとエラーになります。

   <Tabs syncKey="pkg">
     <TabItem label="pnpm">
     ```sh
     pnpm create rune-app .
     ```
     </TabItem>
     <TabItem label="npm">
     ```sh
     npm create rune-app@latest .
     ```
     </TabItem>
     <TabItem label="yarn">
     ```sh
     yarn create rune-app .
     ```
     </TabItem>
     <TabItem label="bun">
     ```sh
     bun create rune-app .
     ```
     </TabItem>
   </Tabs>

2. **コマンドの実行**

   プロジェクトディレクトリに移動し、スターターに含まれる `hello` コマンドを実行して動作を確認します。

   <Tabs syncKey="pkg">
     <TabItem label="pnpm">
     ```sh
     cd my-cli
     pnpm run start hello
     ```
     </TabItem>
     <TabItem label="npm">
     ```sh
     cd my-cli
     npm run start -- hello
     ```
     </TabItem>
     <TabItem label="yarn">
     ```sh
     cd my-cli
     yarn run start hello
     ```
     </TabItem>
     <TabItem label="bun">
     ```sh
     cd my-cli
     bun run start hello
     ```
     </TabItem>
   </Tabs>

   以下のような出力が表示されれば成功です。

   ```
   hello from my-cli
   ```

</Steps>

## 非対話モード

`--yes`（`-y`）フラグを渡すか、TTY のない環境（CI やエージェントなど）で実行すると、対話プロンプトをスキップしてデフォルト値で実行されます。

```sh
npx create-rune-app my-cli --yes

# 現在のディレクトリにプロジェクトを作成する場合
npx create-rune-app . --yes
```

個別の動作はオプションで制御できます。

| オプション | デフォルト | 説明 |
|---|---|---|
| `--yes` / `-y` | `false` | すべてのプロンプトをスキップ |
| `--install` / `--no-install` | `true` | 依存関係のインストール |
| `--git` / `--no-git` | `true` | git リポジトリの初期化 |

```sh
# 依存関係のインストールをスキップ
npx create-rune-app my-cli --yes --no-install

# git の初期化をスキップ
npx create-rune-app my-cli --yes --no-git
```

対話モードでもこれらのオプションを渡すと、該当するプロンプトが省略されます。
