Run the `create rune-app` command and answer a few prompts to quickly scaffold a new Rune project.

## Prerequisites

- [Node.js](https://nodejs.org/) - `v22.12.0` or higher
- Terminal - Rune requires command-line access to initialize a project

## Installation

<Steps>

1. **Create a new project**

   Run the following command in your terminal to create a new Rune project.

   <Tabs syncKey="pkg">
     <TabItem label="pnpm">
     ```sh
     pnpm create rune-app
     ```
     </TabItem>
     <TabItem label="npm">
     ```sh
     npm create rune-app@latest
     ```
     </TabItem>
     <TabItem label="yarn">
     ```sh
     yarn create rune-app
     ```
     </TabItem>
     <TabItem label="bun">
     ```sh
     bun create rune-app
     ```
     </TabItem>
   </Tabs>

   You will be prompted for a project name, whether to install dependencies, and whether to initialize a git repository.

   You can also pass the project name directly as an argument.

   <Tabs syncKey="pkg">
     <TabItem label="pnpm">
     ```sh
     pnpm create rune-app my-cli
     ```
     </TabItem>
     <TabItem label="npm">
     ```sh
     npm create rune-app@latest my-cli
     ```
     </TabItem>
     <TabItem label="yarn">
     ```sh
     yarn create rune-app my-cli
     ```
     </TabItem>
     <TabItem label="bun">
     ```sh
     bun create rune-app my-cli
     ```
     </TabItem>
   </Tabs>

   To scaffold into the current directory, pass `.` instead of a project name. The directory must not contain files that conflict with the template (e.g. `.gitignore`, `package.json`, `tsconfig.json`, or `src/`).

   <Tabs syncKey="pkg">
     <TabItem label="pnpm">
     ```sh
     pnpm create rune-app .
     ```
     </TabItem>
     <TabItem label="npm">
     ```sh
     npm create rune-app@latest .
     ```
     </TabItem>
     <TabItem label="yarn">
     ```sh
     yarn create rune-app .
     ```
     </TabItem>
     <TabItem label="bun">
     ```sh
     bun create rune-app .
     ```
     </TabItem>
   </Tabs>

2. **Run a command**

   Navigate to your project directory and run the starter `hello` command to verify everything works.

   <Tabs syncKey="pkg">
     <TabItem label="pnpm">
     ```sh
     cd my-cli
     pnpm start hello
     ```
     </TabItem>
     <TabItem label="npm">
     ```sh
     cd my-cli
     npm run start -- hello
     ```
     </TabItem>
     <TabItem label="yarn">
     ```sh
     cd my-cli
     yarn start hello
     ```
     </TabItem>
     <TabItem label="bun">
     ```sh
     cd my-cli
     bun start hello
     ```
     </TabItem>
   </Tabs>

   <Aside type="tip" title="npm">
     When using `npm run`, pass the command arguments after an extra `--`.

     ```sh
     npm run start -- hello # Run the hello command
     npm run start -- --help # Show help for the my-cli CLI
     npm run start -- hello --help # Show help for the hello command
     ```
   </Aside>

   If you see the following output, you're all set.

   ```
   hello from my-cli
   ```

</Steps>

## Non-interactive mode

Pass the `--yes` (`-y`) flag to skip all interactive prompts and use default values. This also happens automatically in CI pipelines, agent-driven workflows, and other environments without a TTY.

<Tabs syncKey="pkg">
  <TabItem label="pnpm">
  ```sh
  pnpm create rune-app my-cli --yes

  # Or scaffold into the current directory
  pnpm create rune-app . --yes
  ```
  </TabItem>
  <TabItem label="npm">
  ```sh
  npm create rune-app@latest my-cli -- --yes

  # Or scaffold into the current directory
  npm create rune-app@latest . -- --yes
  ```
  </TabItem>
  <TabItem label="yarn">
  ```sh
  yarn create rune-app my-cli --yes

  # Or scaffold into the current directory
  yarn create rune-app . --yes
  ```
  </TabItem>
  <TabItem label="bun">
  ```sh
  bun create rune-app my-cli --yes

  # Or scaffold into the current directory
  bun create rune-app . --yes
  ```
  </TabItem>
</Tabs>

<Aside type="tip" title="npm">
  `npm create` (and `npm init`) consumes flags itself, so you must separate flags from the package arguments with `--`. Other package managers forward flags directly and do not need the `--` separator.
</Aside>

You can fine-tune the behavior with additional options:

| Option | Default | Description |
|---|---|---|
| `--yes` / `-y` | `false` | Skip all prompts |
| `--install` / `--no-install` | `true` | Install dependencies |
| `--git` / `--no-git` | `true` | Initialize a git repository |

<Tabs syncKey="pkg">
  <TabItem label="pnpm">
  ```sh
  # Skip dependency installation
  pnpm create rune-app my-cli --yes --no-install

  # Skip git initialization
  pnpm create rune-app my-cli --yes --no-git
  ```
  </TabItem>
  <TabItem label="npm">
  ```sh
  # Skip dependency installation
  npm create rune-app@latest my-cli -- --yes --no-install

  # Skip git initialization
  npm create rune-app@latest my-cli -- --yes --no-git
  ```
  </TabItem>
  <TabItem label="yarn">
  ```sh
  # Skip dependency installation
  yarn create rune-app my-cli --yes --no-install

  # Skip git initialization
  yarn create rune-app my-cli --yes --no-git
  ```
  </TabItem>
  <TabItem label="bun">
  ```sh
  # Skip dependency installation
  bun create rune-app my-cli --yes --no-install

  # Skip git initialization
  bun create rune-app my-cli --yes --no-git
  ```
  </TabItem>
</Tabs>

These options also work in interactive mode — any explicitly provided flag will skip its corresponding prompt.
