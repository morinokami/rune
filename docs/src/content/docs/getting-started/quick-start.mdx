This guide walks you through the basics of developing with Rune — adding commands, running tests, and building your CLI — using the starter project created during [installation](/getting-started/installation/).

## Starter project structure

The project generated by `create rune-app` has the following structure:

<FileTree>
- src
  - commands/
    - hello.ts
- tests
  - commands/
    - hello.test.ts
- package.json
- tsconfig.json
</FileTree>

`src/commands/hello.ts` is the only command file included in the starter. In Rune, the file structure under `src/commands` maps directly to your CLI's command structure. See the [Routing](/guides/routing/) guide for details.

## Running and testing the hello command

Let's start by looking at the `hello` command included in the starter. Here is `src/commands/hello.ts`:

```ts
// src/commands/hello.ts
import { defineCommand } from "@rune-cli/rune";

export default defineCommand({
  description: "Say hello from your new Rune CLI",
  run({ output }) {
    output.info("hello from my-cli");
  },
});
```

You define a command by passing an object to `defineCommand()` and default-exporting the result. The `run` function is where your command logic lives. Writing to stdout via `output.info` allows `runCommand` to capture the output during testing.

You already verified this works during [installation](/getting-started/installation/), but let's run it again:

The starter also includes a test file. Let's take a look at `tests/commands/hello.test.ts`:

```ts
// tests/commands/hello.test.ts
import { runCommand } from "@rune-cli/rune/test";
import { expect, test } from "vitest";

import helloCommand from "../../src/commands/hello";

test("hello command prints a greeting", async () => {
  const result = await runCommand(helloCommand);

  expect(result.stdout).toEqual("hello from my-cli\n");
});
```

`runCommand` from `@rune-cli/rune/test` executes a command in-process without spawning a child process, making it easy to assert on its output. Here it verifies that the stdout is `hello from my-cli`. The starter ships with [Vitest](https://vitest.dev/) out of the box, so you can run the tests right away:

## Creating a new command

Now let's create a new command that accepts arguments. We'll build a `count` command that counts up to a given number.

Create `src/commands/count.ts` with the following content:

```ts
// src/commands/count.ts
import { defineCommand } from "@rune-cli/rune";

export default defineCommand({
  description: "Count up to a number",
  args: [
    {
      name: "to",
      type: "number",
      required: true,
      description: "The number to count up to",
    },
  ],
  run({ args, output }) {
    for (let i = 1; i <= args.to; i++) {
      output.info(i);
    }
  },
});
```

Unlike the `hello` command, this one defines a positional argument via `args`. The value is available as `args.to` inside the `run` function with full type safety.

Just by creating the file, Rune automatically recognizes it as a command. Let's try it out:

## Adding an option

Next, let's add a `--reverse` option to the `count` command. Add an `options` definition and update the `run` function:

```diff lang="ts"
  // src/commands/count.ts
  import { defineCommand } from "@rune-cli/rune";

  export default defineCommand({
    description: "Count up to a number",
    args: [
      {
        name: "to",
        type: "number",
        required: true,
        description: "The number to count up to",
      },
    ],
+   options: [
+     {
+       name: "reverse",
+       type: "boolean",
+       description: "Count in reverse order",
+     },
+   ],
-   run({ args, output }) {
-     for (let i = 1; i <= args.to; i++) {
-       output.info(i);
-     }
+   run({ args, options, output }) {
+     if (options.reverse) {
+       for (let i = args.to; i >= 1; i--) {
+         output.info(i);
+       }
+     } else {
+       for (let i = 1; i <= args.to; i++) {
+         output.info(i);
+       }
+     }
    },
  });
```

With `--reverse`, the command counts down instead:

## Testing the count command

Just as we tested the `hello` command, let's write tests for `count`. Create `tests/commands/count.test.ts`:

```ts
// tests/commands/count.test.ts
import { runCommand } from "@rune-cli/rune/test";
import { expect, test } from "vitest";

import countCommand from "../../src/commands/count";

test("counts up to the given number", async () => {
  const result = await runCommand(countCommand, ["3"]);

  expect(result.stdout).toEqual("1\n2\n3\n");
});

test("counts in reverse order with --reverse", async () => {
  const result = await runCommand(countCommand, ["3", "--reverse"]);

  expect(result.stdout).toEqual("3\n2\n1\n");
});
```

The second argument to `runCommand` accepts an array of command-line arguments, so you can simulate real CLI invocations in your tests. Let's run the tests:

## Building your CLI

Build your Rune project to produce a distributable CLI binary.

<Tabs syncKey="pkg">
  <TabItem label="pnpm">
  ```sh
  pnpm run build
  ```
  </TabItem>
  <TabItem label="npm">
  ```sh
  npm run build
  ```
  </TabItem>
  <TabItem label="yarn">
  ```sh
  yarn run build
  ```
  </TabItem>
  <TabItem label="bun">
  ```sh
  bun run build
  ```
  </TabItem>
</Tabs>

The build output is written to the `dist/` directory. The entry point specified in the `bin` field of `package.json` (defaults to `dist/cli.mjs`) can be executed via `npx` or after a global install.
